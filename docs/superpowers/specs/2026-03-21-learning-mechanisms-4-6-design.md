## Executive Summary

Add three intelligence mechanisms to learn.ts: semantic clustering (Louvain community detection on pruned KNN graph), gap detection (L2-normalized topic centroids), and cross-domain transfer (principle-to-insight similarity with zero-tag-overlap surprise filter). Update embed.ts to embed principles. All mechanisms run every time — no threshold gating.

## Mechanism 4: Semantic Clustering (Topic Discovery)

**Function:** `discoverClusters(db, kbRoot) → ClusterFlag[]`

### Algorithm

1. Pull all embeddings from `SELECT id, embedding FROM embeddings` into memory (~330KB at 109 insights)
2. Build pruned KNN graph using `graphology`:
   - For each insight, query top-10 neighbors via sqlite-vec `WHERE embedding MATCH`
   - Create edge ONLY if `distance < 0.25` (similarity > 0.75)
   - Isolated nodes remain isolated
3. Run Louvain community detection via `graphology-communities-louvain`
4. Filter: only flag communities containing insights from 2+ distinct topics
5. For each flagged community, suggest a topic name from the most common shared tags

### Output

`DISCOVER: {N} insights form natural cluster across {topic-a, topic-b} — shared tags: {tag1, tag2}`

### Dependencies

- `graphology` — graph data structure
- `graphology-communities-louvain` — Louvain algorithm

## Mechanism 5: Gap Detection

**Function:** `detectGaps(db, kbRoot) → GapFlag[]`

### Algorithm

1. Group insight embeddings by topic
2. Compute centroid for each topic: `centroid = mean(embeddings)`
3. **L2-normalize the centroid**: `centroid[i] /= ||centroid||` — CRITICAL. Without this, cosine similarity against averaged vectors produces compressed score distributions that miss real neighbors.
4. For each topic centroid, find the 3 nearest OTHER topic centroids by cosine similarity
5. If topic A is very close to topic B (sim > 0.75) but topic B has significantly fewer insights (ratio > 3:1), flag as a gap
6. Cross-reference with tag frequency: find tags that appear in topic A's insights but correspond to no existing topic

### Output

`GAP: You have {X} insights about {topic-A} but {Y} about closely related {topic-B} (shared tags: {tags})`

## Mechanism 6: Cross-Domain Transfer

**Function:** `detectTransfers(db, kbRoot) → TransferFlag[]`

### Algorithm

1. Query principle embeddings: `SELECT id, embedding FROM embeddings WHERE id LIKE 'PRI-%'`
2. For each principle, find top-5 most similar insights in OTHER domains (exclude the principle's own domain)
3. **Surprise Filter**: exclude matches where the principle's tags and the insight's tags have ANY overlap (`intersection.size === 0` required)
4. Only flag if similarity > 0.75 (distance < 0.25)

### Output

`TRANSFER: "{principle title}" ({domain}) may apply to {other-domain}/{topic} (sim: {score}, zero shared tags)`

### Prerequisite

embed.ts must embed principles into the same `embeddings` vec0 table. PRI- prefix ensures no ID collisions with INS- insights.

## embed.ts Update

### Change

Scan `principles/` directory in addition to `domains/` directory. For each principle:
- Hash `title + summary` for change detection (same pattern as insights using `one_line`)
- Generate embedding via Ollama
- Store in the same `embeddings` table with the PRI- prefixed ID
- Mark `embedded: true` in frontmatter

No new tables or schema changes needed — PRI- and INS- prefixes are globally unique.

## Engineering Rules

1. **L2-normalize all centroids** before cosine similarity
2. **Edge pruning**: only connect KNN edges if distance < 0.25
3. **Surprise Filter**: zero tag overlap required for transfer flags
4. **No Ollama calls in learn.ts** — all embeddings come from the database
5. **Run every time** — no threshold gating at current scale
