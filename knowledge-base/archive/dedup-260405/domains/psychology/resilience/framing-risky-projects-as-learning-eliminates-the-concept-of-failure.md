When you frame a project purely as an attempt to create something, failure means total loss. But when you frame it as a way to learn, even complete failure yields knowledge. Lisa Randall suggests treating new things as experiments: if it rules something out, you give up and move on; if there's a way to vary it, you iterate. There is no such thing as failing, since you learn something no matter what.

This reframe is especially powerful when the problem is sharply defined, because failure itself becomes knowledge — if the theorem you're trying to prove is false, or a structural member fails under stress, you've learned something concrete. The practical application is to explicitly articulate what you'll learn from a project before starting it, so that the learning frame is established up front rather than applied retroactively as consolation.
