Investors' core fear with early startups is that founders built something from theory rather than observed need. The antidote is what Sequoia calls a 'proxy for demand' — evidence that real people are doing something painful with inadequate tools right now. The most powerful version is when founders built the product for themselves (Apple, Microsoft, Google, Yahoo all started this way). Next best is stories about people the founder knows personally. This works because specific user narratives are hard to fabricate and easy to evaluate — an investor can immediately judge whether the described pain point is real and whether the solution addresses it. Abstract market sizing ('it's a $5B market') provides none of this signal. The principle generalizes: in any context where you need to prove demand, one vivid specific case outweighs any amount of aggregate data.
