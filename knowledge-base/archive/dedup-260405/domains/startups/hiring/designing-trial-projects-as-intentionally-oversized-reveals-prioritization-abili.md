James Hawkins shared that PostHog designs tasks that are deliberately too much work and open-ended. This is a sophisticated design choice: a perfectly-scoped project only tests execution ability, while an oversized project tests judgment, prioritization, and communication about trade-offs — skills that matter far more in actual day-to-day work.

This principle extends beyond hiring. Linear gives candidates projects that sometimes become their actual first project if hired, meaning the trial doubles as onboarding. Auth0 reused the same exercise across all levels and adjusted expectations, meaning the project itself is a constant while the evaluation criteria scale with seniority. The meta-lesson is that how someone approaches ambiguity and constraint reveals more about their working style than how they execute a well-defined task.
