The prediction is about where capability lives over time. Today, spatial/world understanding is largely external scaffolding — separate 3D-generation tools, video-based world models, simulation engines — wrapped around a language model that doesn't natively understand space. The claim is that this is a transitional architecture: spatial capability will be absorbed into the foundation model itself, yielding a single model that understands both language and the physical/spatial world.

This follows a recognizable pattern in AI: capabilities start as external tools/pipelines (retrieval, code execution, vision) and progressively get pulled into the model as the model grows more capable. The speaker explicitly frames it this way — bring capabilities 'to the model layer instead of becoming pages outside of the model,' so the foundation model can complete spatial tasks itself rather than orchestrating external tools.

Confidence is low here because this is a directional bet by a non-research speaker, and the external-tool vs in-model boundary has historically oscillated (some capabilities stay external for good cost/modularity reasons — e.g. retrieval/RAG persists alongside in-weights knowledge). The falsifiable core: if true, we'd expect leading foundation models to ship native spatial-reasoning/world-model capabilities (not just plug-in tools) within a few model generations. Track whether spatial understanding becomes a native model capability or remains a tool layer — that's the test of this prediction.
