Müller's reflection: choosing open-source for Browser Use was almost arbitrary — 'oh, for Hacker News, okay, it would make sense if it's like open. So we just clicked on GitHub on open. It was like a two-minute decision.' Retrospectively, 'all our traction would be super super hard to get from just pure closed source.' The mechanism: open source creates a community surface for random PRs, fork-marketing (YouTubers featuring it), and trend-coupling ('you can now use DeepSeek with Browser Use' as a hookable announcement). The 'job application AI' demo flopped initially, then went viral 3 weeks later when other YouTubers picked it up. Open source enabled this pickup; closed source would not have.

Application: when evaluating open-source posture for an AI product, frame it as a distribution strategy with measurable surface area, not as ideology. Ask: 'does open-source create more shareable demos, more fork-marketing, more trend-coupling opportunities than closed?' For tooling products in the agent space, yes — community-driven extensions and integrations compound. For commercial-data-layer products (BioRender, Memric), maybe not — the commercial moat depends on closed data + open patterns. Different answers per product type.
