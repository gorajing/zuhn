The source is useful partly because it models epistemic hygiene: it reports a surprising slowdown, lists overgeneralizations it does not support, and links to newer follow-up evidence. That makes it a contradiction source, not a permanent anti-AI-coding conclusion.

The durable lesson is methodological. Zuhn should treat AI-coding productivity as a monitored curve across model/tool generations and task distributions, not as a static belief. Confidence should move less from 'AI coding is bad' and more from 'single-point anecdotes or benchmarks are enough' toward 'only repeated, realistic measurements can update the productivity thesis responsibly.'
