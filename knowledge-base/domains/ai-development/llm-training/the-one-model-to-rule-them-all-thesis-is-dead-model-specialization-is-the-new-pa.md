As recently as 2022-2023, the prevailing belief inside OpenAI was that there would be one model that 'rules them all' — which is why a fine-tuning API seemed unnecessary. This view has completely reversed.

The evidence: Cursor users routinely switch between GPT-5 for planning (max mode), GPT-4o for quick chat, Codex for coding, and specialized composer models for fast first-pass work. OpenAI itself has proliferated models across GPT-4, 4o, 5, o3, Codex, and Sora. The shift isn't bad for OpenAI — it's actually good because it creates a healthier ecosystem, more solutions to provide, and more reasons for developers to stay on the platform. OpenAI has even started opening products like Evals to support other models, recognizing that a rising tide benefits the platform.
