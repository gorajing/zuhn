Tan's framing of GStack: each skill is a distilled, opinionated workflow encoding specific expertise. Office hours captures what 16 YC partners do every day with founders — pushing back on premise, asking 'what's the strongest evidence someone wants this?', exploring 3 alternative approaches, running multi-step adversarial review that auto-fixes detected issues. The skill produces a 6→8/10 design doc improvement automatically before the human even engages.

The operational pattern: rather than generic 'help me think about my idea,' the skill has explicit checkpoints (premise, evidence, alternatives, feasibility, adversarial review) and explicit outputs (design doc with score, issues found and fixed, remaining concerns). The skill has opinions baked in — it pushes back, it ranks options, it autoselects defaults. For builders making Claude Code skills: the temptation is to be flexible and generic; the right move is to be opinionated and specific. For Zuhn specifically: this validates the design of /superpowers and the structured Sleep/Wake/Compress workflows — those are skills in exactly Tan's sense, distilled expertise rather than generic helpers.
