Slash commands are a force multiplier for repetitive complex workflows.
Instead of typing out detailed multi-step instructions every time, pack
them into a slash command that expands into the full prompt. For example,
/dev-docs can expand into instructions to create plan.md, context.md,
and tasks.md with specific formatting requirements. This ensures
consistency (the same detailed instructions every time), saves time
(one command vs. a paragraph of instructions), and reduces errors
(no forgetting steps in a complex workflow). Build slash commands for
any workflow you repeat more than twice.
