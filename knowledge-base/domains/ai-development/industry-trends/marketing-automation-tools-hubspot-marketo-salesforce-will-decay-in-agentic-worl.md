Jason Lemkin's framing of the agentic-era obsolescence: 'I don't think in a year we're going to need any traditional marketing automation software. It's too dated. It doesn't work for agents marketing. There are categories of software where if they don't have a reason to exist in an agentic world, they will go into a terminal state of decay. Agents do not need HubSpot or Marketo or Salesforce or any of these marketing automation tools because they have no need to hand-compose an email in a third-party template.' The structural argument: a marketing automation tool exists because (1) marketers need to compose templates for emails, (2) marketers need to set up sequence triggers, (3) marketers need to A/B test variations, (4) marketers need to track engagement. All four of those tasks are things a human-in-the-middle was doing — slowly, imperfectly, with a tool that helped them be faster. An agent doesn't need any of this. An agent can directly: (a) compose personalized emails per recipient, (b) decide its own sequence triggers based on signals, (c) A/B test continuously without setup, (d) measure and adapt automatically. The marketing automation tool becomes a layer between the user and the agent that adds NO VALUE. The pattern generalizes: any tool whose primary function was 'make humans more efficient at X' faces obsolescence when AI can just DO X. Examples of categories at risk: (1) Marketing automation (HubSpot, Marketo, Salesforce sequencing) — agents compose and send. (2) Helpdesk software (Zendesk, Freshdesk) — agents respond directly. (3) Project management (Asana, Monday.com beyond agentic) — agents coordinate. (4) Bug tracking (Jira) — agents file and resolve bugs. (5) Reporting tools (Tableau, PowerBI) — agents generate reports on demand. (6) Documentation tools (Confluence, Notion for documentation only) — agents write and maintain docs. The corollary for tool buyers: audit your software stack with the question 'in 2027, will agents need this layer between them and the underlying system?' If the answer is no, the vendor is in terminal decay and you should plan migration. The corollary for SaaS founders: if your product's positioning is 'we make X process faster,' identify what happens to your moat when AI just does X. The pivot must happen now or never. The corollary for SaaS investors: the next 18-36 months will produce a SaaS bifurcation. Tools that became agentic infrastructure (provide APIs, surface data to agents, integrate with agent workflows) thrive. Tools that remain human-in-the-loop interfaces wither. Many publicly traded mid-cap SaaS companies are at risk.
