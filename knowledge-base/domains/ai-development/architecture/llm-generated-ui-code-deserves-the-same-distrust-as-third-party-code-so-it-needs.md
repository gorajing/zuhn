The seductive endpoint of generative UI is letting a coding model write the actual frontend at runtime — Casas built a weather agent that hits an API, writes a joke, and emits full HTML/CSS/JS in one tool call with no components and no translation layer. The catch is a trust inversion most builders skip past: 'if we don't trust third-party code, we should not trust code generated by LLMs and then just present it to the user.'

This reframes generative UI as a distribution problem, not just a generation problem. It requires a boundary, containment, and a sandbox. The practical implication: any architecture that ships model-written markup to a user without an execution boundary is shipping arbitrary-code-execution surface. Treat generated UI like a downloaded binary — isolate it before it touches the user's session, DOM, or credentials.
