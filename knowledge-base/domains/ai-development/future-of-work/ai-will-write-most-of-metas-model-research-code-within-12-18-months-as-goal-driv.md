Zuckerberg distinguishes sharply between today's autocomplete (complete this function) and what's coming: an agent you hand a goal, which runs tests, finds issues, iterates, and produces code 'higher quality than the average very good person on the team.' He commits to a concrete 12-18 month timeline for this crossover inside Meta's own Llama research workflow — the place where the team building the models has the most skin in the game and the tightest feedback loop.

Crucially, Meta is not building a general developer tool like Cursor or Claude Code. They're building a coding agent fully plugged into their internal toolchain, optimized specifically for advancing Llama research. This is the 'eat your own dogfood' version of recursive self-improvement — the agents that speed up Llama 5 training are themselves trained by Llama 4. If the timeline holds, by late 2026 the frontier labs will have structurally different cost curves than companies still staffed by human engineers.

The practical implication for builders: if you are relying on human-written code as a moat in the AI-tooling or model-infrastructure space, you have roughly 12-18 months before that moat evaporates inside the labs that set the pace. Plan accordingly.
