Liu's diagnostic of the Sequoia adoption chart: the 50% figure for AI in software engineering is an underestimate of the frontier mode but an overestimate of the deployed mode. Mainstream developers use AI tab-autocomplete (figured out 3 years ago with GitHub Copilot). Frontier developers run something completely different: '30 different Claude Code instances in parallel, each one coupled up to a browser fully autonomous, getting other agents to comment on PRs.' That mode emerged in late 2025 (Karpathy's 'inverted from human-with-AI to AI-with-human').

The operational implication: companies asking 'are we using AI?' often miss that the frontier productivity gain isn't in adoption breadth but in adoption depth. A team that uses Claude Code at GStack-style level 7 outperforms a team where everyone uses autocomplete. For builders making AI tooling: optimize for the frontier user (parallel sessions, PR review automation, browser orchestration) rather than the mainstream user (autocomplete improvements). For Zuhn editorially: 'AI adoption' is a bimodal distribution, not a single curve. Coverage that lumps the two modes together systematically misses where the leverage lives.
