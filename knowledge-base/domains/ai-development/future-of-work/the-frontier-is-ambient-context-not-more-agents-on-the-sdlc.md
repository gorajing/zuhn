Asked whether the SDLC is 'at the end of history' now that agents cover completion, agentic IDEs, orchestration, and security review, Daigle says no — we're in a 'hyper myopic era of AI' where the coding agent loses all the context available across everything a person does outside coding, for boring security and governance reasons. The thing he actually wants is ambient AI: building a feature and having it 'know every spec doc, every email, the conversations I've had online, everything about how this could be implemented, and use that as part of its decision-making.'

The underlying claim is that software development was never a single-lane task — 'it's all the context of the other team members, what the business is doing, what's popular right now.' Once an agent can pull that 'red thread,' it doesn't just write the code the optimal way; it applies 'all of the taste and judgment calls and expertise that I've earned or that we've earned as a group.' This is why GitHub finds the WorkIQ/FoundryIQ context engines so important and why connecting all of a person's data sources (the OpenClaw pattern) is more interesting than yet another way to kick off a coding agent.

The forward-looking implication: differentiation in AI dev tooling will come from context plumbing and the governance that makes it safe at work ('your boss is still not going to get fired' because it isn't leaking private information), not from incremental improvements to the coding agent itself. The hard, unsolved research problem is making the system act the way a specific person or team wants without them having to codify everything by hand.
