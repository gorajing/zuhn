Cherny started Claude Code in late 2024 on the bet that models were 'almost ready' for agentic coding, even though tab-completion typeahead was the state of the art. For the first six months it 'just really didn't work' — barely usable, used for only ~10% of his own code — and even the initial public release was 'not a hit.' Exponential growth only began with Opus 4 in May, then inflected again with each subsequent model (4.5, 4.6, 4.7). The team knew there would be no PMF for six months because they were building for the next model the whole time.

The underlying concept is 'product overhang': the model can already do things that no product has yet captured, and the winning move is to build the harness ahead of the capability curve so you are positioned the moment the model crosses the threshold. The cost is a long valley of no traction that would kill any team optimizing for current-quarter metrics; the payoff is owning the category when capability arrives. This demands genuine conviction about the capability trajectory and the patience to ship something embarrassing for months — a strategy that only pays off if you are right about which capability is imminent.
