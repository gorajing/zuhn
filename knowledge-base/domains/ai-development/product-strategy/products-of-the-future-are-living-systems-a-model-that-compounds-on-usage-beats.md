Today's AI products behave like ordinary static software: the model you used yesterday is the same model making the same mistakes tomorrow, and every correction a user gives it is thrown away. Malde's core thesis is that this is a temporary state — the products that win will be 'living systems' that capture real-world usage and continually update, so intelligence grows the moment you ship to even a few hundred users.

The strategic claim is that this creates a compounding moat. Windsurf's SUI model, Cursor Composer, and Claude Code already build the model around what users actually do, and that loop is what lets them pull ahead. A competitor with a better base model but no learning loop falls behind because it cannot absorb the domain-specific corrections accumulating in production. The advantage widens over time rather than being a one-time capability gap.

For product strategy, this reframes the question from 'which model do I call' to 'what learning loop do I own.' The defensible asset is not the weights at launch but the pipeline that turns your users' corrections into a smarter model every week. It also creates new interface problems — no one has yet designed the product surface for intelligence that gets smarter every day, which is itself an opportunity. Treat the data flywheel, not the initial model quality, as the thing to architect for.
