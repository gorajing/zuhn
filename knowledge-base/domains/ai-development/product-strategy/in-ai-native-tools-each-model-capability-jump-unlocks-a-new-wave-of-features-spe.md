The Cursor team argues that AI programming is unlike previous tech waves because capability improvements arrive continuously with each new model generation. Being even a few months ahead in implementing features for new model capabilities makes your product dramatically more useful. Microsoft with Copilot has done 'fantastic things' but can't innovate at startup speed.

The meta-lesson for AI product builders: the moat isn't a single feature or dataset — it's the speed of the loop between model capability → product feature → user feedback → next iteration. The Cursor team has the same people training models and building UI sitting 18 feet apart, often the same person. This tight feedback loop between research and product is what makes a startup beat Microsoft in their own editor. For any AI product, the strategic question is: how fast can you translate new model capabilities into shipped features?
