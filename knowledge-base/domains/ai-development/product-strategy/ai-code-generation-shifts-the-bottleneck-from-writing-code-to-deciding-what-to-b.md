Mike Krieger reports that 90-95% of code for some Anthropic products is now AI-written, and the transformation happened faster than anyone expected. The bottleneck has shifted from 'can we build it?' to 'should we build it?' and 'can we get it merged and deployed?'

This has profound implications for how teams are structured. Engineering capacity ceases to be the gating factor, so organizations that over-index on hiring engineers while under-investing in product judgment and deployment infrastructure will find themselves building the wrong things faster. The companies that win will be those that can make high-quality decisions quickly and maintain deployment pipelines that can absorb the dramatically increased code output.
