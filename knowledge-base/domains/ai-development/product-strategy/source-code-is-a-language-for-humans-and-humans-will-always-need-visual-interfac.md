Nathan Sobo makes a philosophical argument for the IDE's survival: source code is not binary for machines but a language for human comprehension. Quoting Harold Abelson: 'Programs should be written for people to read and only incidentally for machines to execute.' As long as humans need to understand, review, and verify what AI agents are doing to their codebases, they need interfaces optimized for reading and navigating code.

He speaks from experience: using Claude Code in the terminal, the moment he needs to see context beyond a 10-line excerpt, he reaches for a visual editor. The terminal is great for prompting and delegation but terrible for comprehension. His prediction: the primary interface shifts to prompting (terminal-like), but code editing becomes a necessary secondary interface, and the tools converge. Sometimes the clearest way to express intent is still to write the code directly, especially for simple structures where a natural language prompt would be longer and less precise than the code itself.
