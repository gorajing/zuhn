Imran's framing: setting up a personal agent is becoming '90s tuner-car culture — go find the parts you want, customize, optimize. That's fine, but the trap is mistaking customization for productivity. Many people obsess over agent configuration and never actually do anything with it. The discipline: customization is a means; output is the end. Ask 'what value did I add to my personal life or business life today using this?' not 'what tools did I install today?'

Imran's metric for success: he works at a venture fund and uses Hermes to handle background work, freeing him to talk to 20-30% more founders, get better signal, do better deals. That's the output, not the agent setup. For builders of personal-agent products: design defaults that work out of the box, not configuration surfaces that reward tinkering. For Zuhn editorially: this generalizes — productive AI work is output-measured, not process-measured. The 'mouse and wow' metric Calacanis advocated (engagement growing faster than revenue) is the same principle: what got built, not how nicely the toolkit was configured. For Zuhn specifically: the existing skill design discipline (each skill solves one problem, no skill exists just because) is exactly right. Resist skill proliferation.
