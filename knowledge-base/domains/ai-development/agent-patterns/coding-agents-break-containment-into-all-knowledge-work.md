swyx's central thesis is that he is 'no longer talking about agents for coding' — the same coding agent (Devin, or any open/CLI agent) becomes a general knowledge worker once it has web access and a capable model. He walks through concrete non-coding uses: managing the conference schedule as a source of truth, running ETL syncs against an external vendor's API, researching and contacting speakers, replacing a SaaS CMS, and literally sourcing the physical lobster prop for the event by asking the agent to find vendors and return phone numbers and emails.

The mechanism is that any task you would otherwise hand to an executive assistant or a junior employee can now be done 'serverless, on demand' by an agent. The tool's coding origins are incidental; what matters is autonomy plus web access plus a smart-enough model.

He frames this as a top-three-to-five trend of 2026 and a direct call to action: stop measuring agents by lines of code, and start routing routine knowledge work — forwarding an email and saying 'handle it' — through them.
