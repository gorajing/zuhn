The horizontal rollout playbook for enterprise software (announce, train, measure adoption) systematically underperforms for AI tools because AI's value depends on domain-specific context that horizontal rollout doesn't capture. OpenAI's internal adoption pattern shows the alternative: a small dedicated team per vertical (finance, sales, IT, customer success) who learn the domain deeply, customize the AI tool's UI and harness for that domain, build domain-specific skills (custom prompts, evaluation criteria, memory templates), and only externalize the configuration after it works internally. The implication for enterprise IT leaders deploying AI tools: budget for vertical-specific teams as the unit of rollout, not a single horizontal program. The implication for AI tool vendors: ship customization surfaces (skills, UI modes, agent harnesses) as first-class product features so customers can replicate this pattern; ship one-size-fits-all configurations and you cap your enterprise penetration. The implication for AI consultants and integrators: this is the highest-leverage place for outside expertise — bringing the OpenAI-style vertical-team pattern to organizations that don't have the muscle to build it themselves.
