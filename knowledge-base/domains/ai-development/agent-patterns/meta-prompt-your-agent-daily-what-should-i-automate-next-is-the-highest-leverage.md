Imran's habit for compounding agent value: at the end of every day, ask the agent 'what is one task I'm doing over and over again that I should set up as a cron job?' or 'what should you build for the work I do?' The agent has visibility into your interaction history but doesn't yet proactively surface automation opportunities — so you have to prompt for it.

The time-to-value: it took ~7 days of using Hermes consistently for the agent to know him well enough to start producing useful self-improvement suggestions. The 20-day mark was when the Obsidian dashboard could be auto-organized correctly. The lesson: personal agents have a multi-week ramp-up curve where they get dramatically better, and most users abandon before the inflection. The meta-prompting habit accelerates the curve.

For builders: the right next step for personal-agent products is automating this meta-prompt — the agent should proactively suggest automations based on observed patterns, not wait to be asked. For Zuhn specifically: this is the operational equivalent of the wake.ts briefing — daily structured checkpoints where the agent surfaces what changed, what's stale, and what to focus on. Hermes converging on this pattern is independent validation.
