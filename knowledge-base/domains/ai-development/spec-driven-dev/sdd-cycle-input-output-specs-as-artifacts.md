Spec-Driven Development defines a full cycle: input specs (PRD, tech brief,
requirements) feed into the workflow (research, plan, code, review), which
produces output specs (research logs, code notes, findings). The key
distinction from traditional development is that output artifacts — not
just the code — are treated as first-class deliverables. This means
research logs, decision records, and findings are as important as the
code itself, because they capture the reasoning that produced the code.
