The reframe is that agents already have their own currency — tokens — and every interaction in a coding harness or any LLM application is in effect spending money, whether proxied through a subscription or converted from input/output tokens into dollars. We already let agents spend; we just only let them spend with their LLM provider.

The open question is therefore not whether agents will transact but how to enable other currencies, spend patterns, payment methods, and business interactions. This shifts the framing from speculative ('what if agents could buy things') to infrastructural ('agents are economic actors; build the rails'). Treating it as present-tense changes prioritization: the work is plumbing and safety, not proving demand.

Marked as inspiration/framing because its value is in reorienting how you think about the agent economy rather than prescribing a specific action — but it directly motivates the deterministic-payment and scoped-mandate work that follows from taking agents seriously as buyers.
