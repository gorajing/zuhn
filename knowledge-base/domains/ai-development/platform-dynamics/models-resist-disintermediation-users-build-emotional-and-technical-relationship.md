The conventional wisdom was that models would be interchangeable like cloud infrastructure — swap one for another behind an abstraction layer. This hasn't happened, for two reasons:

1. Emotional: Users form genuine relationships with model personalities. The GPT-5 personality change created real dissonance. People feel the difference when their model changes — this is 'anti-disintermediation technology' that you almost have to expose directly to users.
2. Technical: Developers who build products around specific models (iterating on harnesses, tool configurations, prompt patterns) create deep technical coupling. The retention on OpenAI's API has been 'surprisingly high' even when tools exist to swap models.

This has profound implications for defensibility: unlike prior platform cycles where the infrastructure could be abstracted, the model IS the product experience, making it inherently sticky.
