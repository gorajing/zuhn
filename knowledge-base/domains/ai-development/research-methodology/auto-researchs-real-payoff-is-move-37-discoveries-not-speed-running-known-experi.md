Every AI wave has its AutoML moment, and agentic 'auto-research' is this wave's version. There are two qualitatively different things it can do. The first is speed-running: the coding agent autonomously runs the experiments and ablations you would have run yourself while you sleep — genuinely useful, but it's automating known work and exploring paths you'd already have chosen. Much of research is exactly this kind of empirical 'move the pieces around and see what works,' which is closer to engineering than to deep architecture design.

The second, far more valuable mode is when the agent shoots off trajectories a human wouldn't have considered — and they work, producing a genuine discovery. That's the 'Move 37' threshold (AlphaGo's famously non-human move). It requires the system to explore beyond the hypothesis space a researcher would have enumerated, not just execute within it faster.

For anyone building or evaluating research-automation tooling, this is the discriminating question: does it merely compress the cost of experiments you'd have run, or does it expand the set of experiments that get run at all? The former is a productivity win; the latter is a discovery engine. A related near-term shift Sanseviero predicts: the next generation of fine-tuners won't write training code at all — they'll prompt agents using prepackaged skills to kick off experiments — which makes the 'speed-running' mode broadly accessible while the genuine-discovery mode stays the open frontier.
