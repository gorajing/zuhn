Hopkins observes that enterprises are deploying voice agents at massive scale and more rapidly than any other agent category, and the reason is infrastructural rather than technological. Customer service already has standard operating procedures, IVR trees, and documented call flows, so the conceptual leap from a scripted human call to an autonomous voice agent is far smaller than, say, a financial-services agent making decisions on your behalf. The scaffolding for the new capability is already in place.

The deeper pattern is a platform-shift curve identical to web and mobile: people first put existing things (a brochure on the web, a website on a phone) into the new medium, then discover natively new experiences (web apps, mobile apps). Enterprises start by automating the obvious phone workflows (support, logistics) and then realize voice unlocks novel use cases — concierges that drive product discovery, upsell, and adoption — that they would never have scoped otherwise.

The actionable implication: when evaluating where an agent modality will land first, look for the domain where the *surrounding infrastructure and procedures already exist*, not where the model is most capable. Adoption speed is gated by how small the remaining leap is, and the first wedge use case is almost never the eventual large market.
