Kimi's Sarah argued that openness matters more in the agent era than in the chatbot era. Chatbots optimized a single output — the final answer. Agents, by contrast, involve multi-step workflows where accuracy compounds across tool calls, reasoning chains, and action sequences. Enterprises need fine-grained control over each step: which tools are called, how errors are handled, where to optimize latency versus accuracy.

This creates a structural advantage for open models in agentic use cases. Closed APIs provide a black box — acceptable for chat but problematic when you need to inspect, modify, or optimize intermediate agent steps. Open models let enterprises customize the full stack: fine-tune for specific tool-calling patterns, optimize inference for their particular agentic workflow shape, and maintain data privacy. The Alibaba Cloud panelist reinforced this by noting that low-latency and data-privacy scenarios make open source 'the only choice.'
