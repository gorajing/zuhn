Intercom's vision was to get the agent to 'act like a senior engineer on any technical task' and to 'connect it to everything' an engineer can do on their laptop. That requires onboarding it exactly like a human hire: Rails conventions, architecture, React patterns, testing standards, security rules — the agent 'absolutely has to know the Intercom-specific information to be able to do the job.'

The mechanism is a flywheel: when the agent gets something wrong, hits an issue, or goes down the wrong path, you update the guidance — and everyone contributes. They encode this knowledge in skills, guidance documents, and hooks that force correct behavior, and even bypass standard update mechanisms to push internal plugins to hundreds of laptops. The unlock that makes 'connect it to everything' safe isn't recklessness — it's that a mature company already has the controls, permissions, and audits that let them unleash the agent the same way they unleash human engineers.
