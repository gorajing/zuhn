Peter Steinberger made OpenClaw deeply self-aware: the agent knows its own source code, understands how it sits in its harness, knows where documentation lives, knows which model it runs, and whether voice or reasoning mode is active. This self-awareness wasn't just for transparency — it enabled emergent self-modification.

When users prompted the agent and it didn't like something, it would just modify its own software. Steinberger also used self-introspection for debugging: 'Hey, what tools do you see? Can you call the tool yourself? Read the source code. Figure out what's the problem.' The agent debugging itself became the natural development workflow. This approach also democratized open-source contribution — people who had never written code could submit 'prompt requests' (PRs generated by the agent), making their first open-source contribution. The principle: agent self-awareness is a force multiplier for both development velocity and accessibility.
