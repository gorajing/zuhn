Google DeepMind's Rabin shared his personal workflow: he uses multiple coding agents even within the Google stack, each specialized for different tasks — some for ideation, some for fixing unit tests, some for other purposes. He noted they each excel in different areas, and his workflow changes roughly every few months as models rapidly improve.

This matches an emerging pattern: rather than waiting for one perfect coding agent, productive practitioners compose workflows from multiple specialized tools. The key insight is that the best current approach is tool pluralism, not tool monogamy. Rabin's prediction that his workflow will have 'changed five times' by year-end suggests the developer tools landscape is still in rapid flux, and the winning strategy is staying flexible and continuously re-evaluating which agent is best for which task.
