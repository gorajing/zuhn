The Atlassian CEO makes a compelling case against the 'vibe code your own Workday' narrative using David Ricardo's 1817 theory of comparative advantage: even if you could vibe code your own HR system, you should still buy one because the edge cases buried in enterprise software (like Indiana maternity leave rules) represent decades of accumulated knowledge embedded in deterministic rules that aren't exposed or easily replicated.

However, vibe coding creates massive value in extensibility: custom applications built on top of existing platforms for specific use cases. Example: a Miami office needs a conference room booking app that integrates with Workday's HR policies, used by only 20 people. Previously too expensive for IT to build, now vibe-codeable in hours. This uses Workday's data and rules underneath but provides a custom interface.

This insight reframes vibe coding from 'SaaS killer' to 'SaaS enhancer' — it makes existing platforms stickier by enabling the long tail of customization that was previously too expensive to build. OpenClaw users building personal apps on top of Gmail exemplifies this: they're not replacing Gmail, they're extending it.
