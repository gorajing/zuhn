Adora warns against the 'Frankenstein approach' where founders talk to users, collect feature ideas, and rush to build all of them. The resulting product becomes a patchwork monster that doesn't coherently solve any problem well.

The fix is to treat every feature request as a symptom, not a diagnosis. When a user says 'add X,' the real question is: what problem are they trying to solve, and is X actually the best solution? Often users are working around a deeper issue — either a problem the product created, or a prerequisite that must be met before they'll pay. Building the requested feature hides the root problem under more complexity. This discipline is especially important pre-product-market-fit when every engineering hour matters.
