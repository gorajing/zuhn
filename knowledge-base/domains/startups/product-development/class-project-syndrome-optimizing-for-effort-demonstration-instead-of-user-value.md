Graham identifies two missing elements in class projects that distinguish them from real startups: iterative problem discovery and intensity. In school, proving your initial plan was wrong earns a bad grade, so students build every line of code toward demonstrating effort. In startups, the first six months of building often exist solely to prove the initial idea was mistaken—and that's extremely valuable.

This creates a measurable behavioral difference. Class-project builders want every line of code to count toward a final deliverable. Startup builders need to treat most early code as disposable exploration. The market's scoring function is brutally different from academia's: users want software that does what they need, full stop. There is no partial credit for a 'good effort.' Graham argues this is one of the most important lies that education implants and that the real world must beat out of you.
