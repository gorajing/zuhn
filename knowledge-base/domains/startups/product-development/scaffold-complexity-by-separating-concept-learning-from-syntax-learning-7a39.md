The tutorial's 6-project arc deliberately escalates: Project 1 (payroll) teaches cell references and basic formulas. Project 2 (gradebook) reuses formulas but adds conditional formatting. Project 3 (decision matrix) reuses formatting but adds weighted scoring. Each project introduces exactly one new layer while recycling the mechanical skills from previous projects.

This scaffolding principle applies directly to product onboarding: if your first-run experience requires users to simultaneously learn your navigation model, your data model, and your core workflow, most will bounce. The fix is sequencing — let users succeed with familiar patterns first, then introduce novelty one layer at a time. Products that feel 'intuitive' are rarely simple; they're well-scaffolded, ensuring users never face more than one unfamiliar element per interaction.
