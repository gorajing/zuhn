Graham contrasts Java applets (heavily promoted by Sun, ignored by hackers, died) with Ajax/Google Maps (no marketing push, hackers spontaneously built Busmonster and thousands of mashups). The pattern is clear: when you find PR firms promoting something as the next development platform, you can be sure it isn't one.

This is a reliable litmus test for platform viability. Real platforms generate organic developer activity that's visible before any official launch or marketing campaign. The corollary for builders is that developer adoption metrics (GitHub stars, Stack Overflow questions, indie projects) are far more predictive than press coverage or conference buzz. If you're building a platform and need to convince developers to use it, something is fundamentally wrong with the value proposition.
