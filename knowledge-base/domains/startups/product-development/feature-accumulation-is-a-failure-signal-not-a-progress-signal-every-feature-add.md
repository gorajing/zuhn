Tim's team built 20 games, multiple features, a complex mobile app — all before getting real user feedback. Every feature added created more code to maintain, more surface area for bugs, and more distance from the core value proposition. The instinct to 'make it better by adding more' is backwards.

This challenges the common startup narrative of feature-driven growth. The counterweight principle: if your first feature doesn't retain users, your 20th feature won't either. Simplicity is not just an engineering preference — it's a diagnostic tool. With one feature, you know exactly what users are rejecting. With twenty, you can't tell what's failing.
