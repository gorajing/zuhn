Cherny calls latent demand 'the single biggest principle in product' and says every feature after the initial CLI was built by reading it. The principle: people will only do a thing they already do. If they're trying to do something and you make it easier, that's a good idea; if you try to make them do a different thing, they won't. CLAUDE.md emerged because users were already writing markdown files and feeding them to the model. Plan mode emerged because users were already asking Claude to 'plan this out but don't write code yet' — he saw it in GitHub issues and internal Slack, wrote it in 30 minutes one Sunday night, and shipped it Monday.

The method is observational, not visionary: watch what users hack together, stand behind them while they work, read the issue tracker for the behavior they're forcing. The signal that latent demand exists is users 'jumping through hoops' — e.g. non-technical Anthropic staff installing a terminal tool just to use the agent. Build the path they're already cutting.
