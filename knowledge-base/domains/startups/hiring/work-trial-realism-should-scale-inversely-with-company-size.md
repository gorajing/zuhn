Gumroad represents one extreme: hiring candidates as contractors for 4-6 weeks with full access to internal tools, essentially operating on a 'hired by default, fired in 30-60 days if underperforming' model. Linear runs 3-5 day co-working trials in the actual codebase. Both can afford this because they're small and each hire matters enormously.

As companies grow, this approach breaks down. PostHog moved from actual work to representative work because the codebase became too complex for onboarding trial candidates, and actual work created IP ownership complications. Automattic shifted from real codebases to forked smaller projects for consistency and scalability. Auth0 ultimately shortened their exercises after the Okta acquisition, explicitly accepting less signal. The pattern suggests a company should start with maximum-realism trials and consciously trade signal for efficiency only as hiring volume demands it.
