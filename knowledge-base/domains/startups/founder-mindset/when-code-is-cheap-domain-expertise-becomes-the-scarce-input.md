Cherny's frame is the printing press: before it, roughly 10% of Europe was literate, often employed by illiterate lords whose job was to read and write for them; afterward the cost of a book fell about 100x and literacy eventually reached ~70%, yet professional writers still exist. Software is following the same arc, only much faster than fifty years — building software is becoming a universal skill 'like sending a text message,' fully democratized so anyone can do it.

The consequence is an inversion of who holds the advantage. When coding is the easy part, the bottleneck moves to domain knowledge. The best author of accounting software is a great accountant, not an engineer, because they understand the domain that the code merely encodes. For founders and teams this reframes hiring and team composition: embedded domain experts who can now build directly may out-compete generic engineering talent operating in unfamiliar domains. It predicts a wave of vertical, founder-is-the-expert software where the moat is domain depth rather than technical execution.
