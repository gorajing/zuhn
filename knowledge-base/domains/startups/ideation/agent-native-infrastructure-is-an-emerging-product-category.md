Human-facing services are deliberately hostile to automation: Gmail and every email provider make it as hard as possible for bots to sign up and operate, specifically to prevent spam. Rather than trying to make an agent operate a human email account (which the panel calls both hard and 'sketch'), Agent Mail built the first email provider designed for AI agents — and it exploded once Open Claw adoption took off. The pattern generalizes: there is room for a parallel tech stack 'from agents for agents.'

This reframes a whole class of startup ideas as 'build the agent-native version of X.' The open question raised — 'has anyone built Twilio for agents / phone numbers for agents yet?' — is presented as a request for startups. Any human-first primitive (identity, payments, phone, calendar, reservations) that agents currently have to hack around is a candidate for an agent-native rebuild where automation is a feature rather than an attack to be blocked.
