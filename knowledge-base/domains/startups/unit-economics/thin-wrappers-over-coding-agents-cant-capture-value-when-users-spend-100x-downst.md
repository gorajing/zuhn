Knight-Webb shut down Vibe Kanban live on stage despite strong traction (30,000 monthly active users, 25,000 GitHub stars) because the unit economics were unworkable. The product was an orchestration layer — a button that helps you run Codex or Claude Code in parallel — not a coding agent itself. Users would pay roughly a $30 subscription and then press a button that helped them spend $3,000 with the underlying model provider, so the company captured a tiny sliver of the value it enabled. His diagnosis: in the current market, the only people making money are doing one of two things — selling to enterprise or reselling tokens — and Vibe Kanban did neither, serving individuals, startups, and small companies. The transferable lesson is that traction (users, stars) is not value capture; if your pricing model sits orthogonal to where the money actually flows, scale only deepens the loss. Pivoting up-market into enterprise was possible but, in a maturing, crowded market, 'it's no fun playing for eighth place.'
