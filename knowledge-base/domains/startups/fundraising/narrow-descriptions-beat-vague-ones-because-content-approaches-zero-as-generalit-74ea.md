When founders try to capture every possibility their product might evolve into, the description becomes so general it communicates nothing. Paul Graham compares this to a calculus limit: as you approach a description of 'anything,' the information content approaches zero. Investors will discard vague statements like 'a system to collaboratively leverage the value of information' as boilerplate.

The fix is to treat your pitch like incremental development: start with a gripping but perhaps overly narrow description ('an easy web-based database'), then flesh it out. At every point you should have a 'working description' in the investor's head—the verbal equivalent of working code. This principle extends beyond fundraising to any high-stakes communication where the audience has limited attention and high filtering.
