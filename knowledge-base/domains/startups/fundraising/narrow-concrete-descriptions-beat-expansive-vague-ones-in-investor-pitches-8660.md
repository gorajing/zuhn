Paul Graham observes that as a startup description approaches something that 'could be anything,' its information content approaches zero—a calculus-like limit. Investors hearing 'a system to collaboratively leverage the value of information' will mentally discard it and wait impatiently for the actual explanation. The counterintuitive fix is to start with a description that feels misleadingly narrow ('an easy web-based database') and then flesh it out. This mirrors incremental development: always have a 'working description' in the investor's head, even if incomplete. The narrow version is gripping and memorable; the expansive version is forgettable noise.
