Paul Graham observes that as a product description approaches 'could be anything,' its information content approaches zero. Investors discard sentences like 'a system to collaboratively leverage information' as meaningless boilerplate. The counterintuitive fix is to start with a description that feels too narrow — 'an easy web-based database' — then expand. This mirrors incremental development: always have working code (a working mental model in the investor's head). The narrow description gives investors a concrete anchor to attach subsequent details to, while the vague one gives them nothing to hold onto.
