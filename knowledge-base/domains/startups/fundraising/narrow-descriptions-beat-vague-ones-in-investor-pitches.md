Founders resist narrow descriptions because early-stage startups could become many things. But as Paul Graham notes, as a description approaches 'could be anything,' its informational content approaches zero. Investors will discard vague language like 'collaboratively leverage the value of information' as meaningless boilerplate.

The better approach mirrors incremental development: start with a gripping but perhaps overly narrow description, then flesh it out. The goal isn't to describe everything your system might become—it's to create a 'working description' in the investor's head that compels them to want to learn more. Successive approximation beats comprehensive vagueness.
