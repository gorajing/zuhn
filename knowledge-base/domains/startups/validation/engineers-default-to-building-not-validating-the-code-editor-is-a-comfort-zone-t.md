Tim's team raised money, started engineering immediately, and built an app with 20 different games before getting meaningful user feedback. The code editor is seductive because every line of code feels like measurable progress. But this is a trap: building without validation means you're optimizing something nobody asked for.

The counterweight to 'ship fast' advice: shipping fast is useless if you haven't validated first. Tim's retrospective is clear — he would spend months NOT writing code, instead doing user interviews and even trying to get pre-purchases before building anything. The right sequence is validate -> build minimal -> ship, not build -> ship -> hope.
