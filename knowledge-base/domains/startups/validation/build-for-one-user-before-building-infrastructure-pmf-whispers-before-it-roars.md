Substack's origin story is a masterclass in minimal viable validation. Chris Best didn't build a platform and hope writers would come — he found one writer (China expert Bill Bishop), manually set everything up, and proved the economic model worked before writing a single line of scalable code. Bishop made six figures on day one.

This approach eliminates the most common startup failure mode: building elaborate infrastructure for a product nobody wants. When your 'product' is literally typing a user into a database by hand, you can't hide behind features — the value proposition has to stand on its own. The manual work that doesn't scale is actually the point: it forces founders to understand their user's problem at the deepest possible level before abstracting it away.
