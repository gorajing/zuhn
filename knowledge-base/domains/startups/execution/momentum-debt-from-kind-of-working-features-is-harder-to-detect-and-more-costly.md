Broken features are easy to identify and scrap. The real momentum killers are features that 'kind of work'—the integration built for a prospect who never converted, the dashboard a few users occasionally check, the side project still 'gathering data' after three months. Each one adds friction: cognitive load for the team, complexity for new features, and confusion for users. They're organizational scar tissue.

The antidote is treating deprecation as a first-class momentum tactic. Kayako's recovery to $2M ARR came not from tweaking a broken pricing model but from tearing it down entirely. Instagram's inflection point was killing everything except photo sharing. Running a quarterly feature audit—benching anything that isn't actively accelerating growth—converts accumulated drag into recovered velocity. The courage to abandon what isn't compounding is itself a competitive advantage.
