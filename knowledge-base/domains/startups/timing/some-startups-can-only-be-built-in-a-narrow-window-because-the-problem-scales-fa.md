When Larry and Sergey started back rub in 1996, the web had around 600,000 sites and was growing 723% year-over-year. That meant two Stanford students could, with borrowed computers from loading docks and Stanford's network bandwidth, actually copy a meaningful chunk of the internet and compute page rank on it as a research project. Had they started the same project denovo in 1998 or 1999, the storage, compute, and bandwidth costs of building a full index from scratch would have been prohibitive for any unfunded team.

This creates a specific pattern: for infrastructure-heavy ideas where the cost of building scales with the problem you're indexing, there is typically a narrow window when the problem is big enough to be valuable but small enough to be tractable from zero. Miss the window on either side and you can't start — either the idea isn't worth pursuing yet, or the incumbents have such a head start on capital-intensive infrastructure that competing requires hundreds of millions rather than hundreds of thousands.

For founders evaluating ambitious ideas: don't just ask 'is this a good idea?' Ask 'is the window for bootstrapping from zero still open?' Crawling the web, building a social graph, indexing code, or training frontier models have all had windows that opened and closed. If the window is closing, either move immediately or accept that the idea now requires an entirely different financing and team structure than it would have a year earlier. For investors: the window argument is why early bets in emerging infrastructure categories can be massively undervalued — the option to start cheaply expires.
