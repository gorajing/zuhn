Graham argues that simplicity should be the default but rarely is, because creators instinctively add ornament to mask the absence of substance. Beginning writers adopt pompous tones, designers resort to swooshes, painters discover they're expressionists — all evasion. When you strip away decoration, you're forced to deliver substance.

This principle applies directly to product development and software architecture. A complex API with many configuration options often masks an unclear mental model. The discipline of simplicity — fewer options, shorter proofs, less code — is a forcing function that reveals whether you've actually solved the problem or just decorated around it. The test isn't whether something looks minimal, but whether removing anything would break it.
