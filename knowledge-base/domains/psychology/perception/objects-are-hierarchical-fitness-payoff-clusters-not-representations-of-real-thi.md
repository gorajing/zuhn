When Yale researchers challenged Hoffman's fitness-beats-truth result by showing that with 20,000+ payoff functions, truth-seeing organisms could evolve, Hoffman's team found a crucial flaw: the Yale simulations didn't allow organisms to create objects. When organisms could hierarchically cluster similar fitness payoff functions into data structures (what we call 'objects'), truth-seeing organisms were once again eliminated.

This reveals what objects actually are from an evolutionary perspective: a bottle is not a perception of a real bottle but a data structure encoding a cluster of action-payoff relationships (drinking, weaponizing, etc.). An apple encodes a different cluster. Each perceived object represents a different set of possible actions and their fitness consequences. This reframes perception not as a window into reality but as an efficient action-oriented interface — we see 'objects' because clustering payoff functions into compact data structures is computationally cheaper than processing raw fitness landscapes.
