The video highlights that Photosystem II is confusingly named because it was discovered second but acts first in the light-dependent reactions. This pattern — where historical naming obscures logical sequence — appears throughout technical fields (JavaScript's prototype chain, HTTP status codes, CPU ring 0 being the most privileged).

The pedagogical insight is that explicitly acknowledging the mismatch between discovery order and logical order reduces cognitive load. Rather than pretending confusing names make sense, effective teaching calls out the historical accident directly. Learners can then file the name as arbitrary convention rather than wasting mental energy trying to find meaning in the numbering. This applies to documentation, onboarding materials, and API design — when naming is counterintuitive, explain why rather than hoping users won't notice.
