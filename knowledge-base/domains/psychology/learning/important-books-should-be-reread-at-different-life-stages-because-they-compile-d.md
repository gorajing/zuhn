Graham identifies a cognitive error: treating reading like carpentry, where doing something again implies you did it wrong the first time. But if knowledge is compiled into your mental model using the current state of your brain, then the same source material will produce different outputs at different life stages. A book on leadership read before you've led anyone compiles very differently from the same book read after a decade of management.

This has a direct implication for how to build a personal reading program: a small library of high-signal books revisited periodically will compound in a way a large library read once will not. The re-reading is not review — it is a genuinely new compilation pass over new hardware.

The analogy to software is precise: just as recompiling the same source code on a newer architecture can produce a faster binary, rereading the same book with a more developed mind produces richer, more integrated knowledge.
