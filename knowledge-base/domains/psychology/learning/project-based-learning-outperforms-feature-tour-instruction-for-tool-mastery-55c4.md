The tutorial teaches Excel entirely through 6 escalating projects — payroll calculation, gradebook, career decision matrix, sales database, car inventory with CSV/VLOOKUP/pivot tables, and challenge problems. Each project introduces new features only when they solve a concrete problem the learner already cares about.

This mirrors research on situated cognition: knowledge acquired in context transfers better than knowledge acquired in abstract isolation. When a learner encounters VLOOKUP while trying to match car VINs to prices, they grasp both the syntax and the *reason* for the function simultaneously. Feature-tour instruction (click File, then click Format, then click...) creates recognition memory but not procedural fluency. The project-based approach creates procedural fluency from the first session because every feature is immediately applied to a meaningful output.
