The tutorial deliberately starts with just a text editor and browser — no build tools, no frameworks, no package managers. This isn't just simplification; it's a cognitive load management strategy. Every additional tool in a learning environment is a potential source of errors that have nothing to do with the subject being taught. A webpack configuration error when you're trying to learn HTML tags creates confusion about what's 'HTML' vs what's 'tooling.'

This principle applies broadly to onboarding in any domain: strip the environment to the minimum viable toolchain, let the learner build confidence with core concepts, then layer complexity incrementally. The mistake most curricula make is front-loading professional-grade tooling that experts find convenient but beginners find overwhelming. Graduated complexity — starting bare-bones and adding tools only when the learner feels the pain they solve — produces deeper understanding than starting with the 'right' setup.
