Vim's modal editing — where the same keys perform different actions depending on the current mode — is initially confusing but enables operations like 'delete 3 lines' (3dd) that are impossible to match with mouse-based editing. The 50-year survival of this paradigm despite far more intuitive alternatives proves that memorization-heavy interfaces can be durably superior for expert users.

This principle extends beyond editors: any tool with a steep learning curve but high skill ceiling (Blender, Emacs, terminal workflows) follows the same dynamic. The lesson is that 'user-friendly' and 'productive' are not synonyms — the most productive tools often require the most investment to learn. The key factor is whether the user will spend enough time with the tool to recoup the learning investment.
