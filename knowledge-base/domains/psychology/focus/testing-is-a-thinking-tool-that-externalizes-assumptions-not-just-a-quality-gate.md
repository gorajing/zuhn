CS50 Python dedicates an entire lecture to testing with pytest — unusual for an introductory course. The key insight isn't that testing catches bugs (obvious), but that writing a test forces the programmer to answer 'what do I actually expect this to do?' before they can assert anything. This is a metacognitive exercise: you must make implicit assumptions explicit, consider edge cases you hadn't thought of, and define the boundary between correct and incorrect behavior.

This reframes testing from a chore that follows coding into a thinking tool that should precede or accompany it. The same principle applies far beyond programming: any domain where you can formalize your expectations and check them against reality benefits from this discipline. In the AI era, this is especially relevant — testing AI outputs requires the same skill of articulating what 'correct' means before you can evaluate whether the model delivered it. The ability to write good tests is really the ability to think precisely about desired outcomes.
