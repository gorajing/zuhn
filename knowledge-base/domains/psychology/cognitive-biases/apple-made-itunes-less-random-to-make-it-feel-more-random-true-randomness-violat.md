When Apple's iTunes shuffle used true randomness, users complained that songs from the same artist appeared consecutively — which is statistically expected from random sequences. Steve Jobs responded by making the shuffle 'less random to make it feel more random,' artificially spacing similar songs apart.

This reveals a fundamental design principle: human perception of randomness is systematically biased. We expect random sequences to alternate more than they actually do (the gambler's fallacy at scale). Any product that generates random output — recommendation algorithms, A/B test assignments, card games, playlist shuffles — must account for this gap between statistical randomness and perceived randomness. The implication extends beyond UX: when evaluating whether a pattern in data is meaningful, our default intuition about what 'random' looks like is unreliable, and we need statistical tools rather than gut feeling to distinguish signal from noise.
