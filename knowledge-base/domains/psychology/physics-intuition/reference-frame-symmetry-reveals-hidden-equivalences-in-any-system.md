When the team observes the baseball stripping leather off the glove, they make a key insight: from the ball's reference frame, the glove is traveling at supersonic speed toward it. This immediately explains the damage pattern — it's equivalent to hitting a stationary ball with a glove at Mach 1.35. The physics is symmetric; only our perspective assigns 'moving' and 'stationary.'

This reference frame shift is a powerful general-purpose mental model. In competitive strategy, asking 'what does this look like from the competitor's perspective?' often reveals dynamics invisible from your own frame. In negotiation, shifting to the other party's reference frame reveals their constraints and motivations. In debugging, asking 'what if the bug isn't in my code but in my assumptions about the input?' is a reference frame shift. The key principle: when an interaction is confusing, try swapping which element you consider fixed and which you consider moving.
