Hours spent on a detailed spec before coding caught issues before they became bugs, with zero scope creep across 5 phases. Babbage's Rule showed that parallel calculation with the same method produces the same errors — driving computing to prioritize verification over speed. The same principle applies to software: the cheapest bug fix is the one you never write because the spec made the error visible before implementation.
