C endures not because it's the best at any one thing, but because it occupies a unique position bridging high-level developer ergonomics and low-level hardware control — no replacement can match both simultaneously. Fortran's expression/statement distinction survived decades past its punch-card origin because historical accidents become load-bearing constraints. The pattern is general: systems that survive long-term do so by minimizing their core axiom set (fewer primitives = fewer breaking changes), while systems that grow by accumulating features create deep inheritance hierarchies and refactoring traps that compound over time. Libraries are reusable because they are language, not because they are object-oriented.
