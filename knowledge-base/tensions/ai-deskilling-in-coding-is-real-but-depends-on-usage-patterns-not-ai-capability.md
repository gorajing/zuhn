**Side A:** "AI deskilling in coding is real but depends on usage patterns not AI capability itself"
**Side B:** "AI programming tools have a unique competitive dynamic where every model improvement makes them better for everyone"
**Reason:** One claims AI degrades human coding skills, the other claims AI tools universally improve coding — these are opposing views on whether AI helps or hurts developer capability
**Resolution:** These stances are not actually opposed -- they describe different levels of the same system. Side B (model improvements help everyone) describes the tool layer: better models produce better autocomplete, debugging, and code generation for all users. Side A (deskilling depends on usage patterns) describes the human layer: whether those improvements degrade or enhance human skill depends on how the person engages with the tool. The key variable from Anthropic's own research is usage pattern: using AI as a verification partner (write code, then check AI suggestions) preserves skill; using AI as a generation engine (accept AI output, skip understanding) degrades it. Better models accelerate both skill enhancement AND skill degradation depending on the user's approach.

**Prediction:** Within 2 years, developers who primarily use AI for code review and debugging (verification pattern) will maintain or improve coding assessment scores, while those who primarily use AI for code generation (delegation pattern) will show measurable decline, regardless of which model version they use.
